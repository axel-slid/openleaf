Run locally

Clone the public repo, install dependencies, then launch the Electron app from the project directory.

git clone https://github.com/axel-slid/openleaf.git
cd openleaf
npm install
npm start

Project workflow

Use New Project for a blank paper, TeX entry file, folder, zip, tar, tar.gz, or tgz archive. The file toolbar can create files, create folders, upload files or folders into the current project, and export the whole project package.

GitHub push

Push copies only LaTeX source files into the dedicated public documents repo at github.com/axel-slid/openleaf-latex-documents. It does not upload generated PDFs, figures, datasets, caches, or the full project folder unless those files are LaTeX source files.

Remote workspace

Use New SSH Project on the Projects page, choose a known host or enter a new one, then set the server path. Openleaf opens an SSH terminal at that path so you can git clone, pull, compile, or inspect the remote project like a VS Code remote workflow.

ssh user@host.edu
cd /path/to/project
git pull
tectonic main.tex

AGENTS.md

The AGENTS.md settings page lets you write project instructions for AI-assisted edits. Put build commands, style rules, file ownership notes, and safety constraints there. Save it once, and Openleaf keeps it in the project root so Codex or other agents can read the same context.

Command palette

Press Cmd+P to open commands. Useful commands include /open, /templates, /settings, /compile, /history, /vim, /minimap, and /relative. Type a project name after /open to jump directly to it.

Editing and review

Use Code for source editing and Visual for structured manuscript edits. Cmd+F highlights every match in the current text. If Vim is enabled, the mode indicator appears next to Code and Visual. History shows session edits inside the app so you can review changes without leaving the workspace.

PDF and packages

Compile PDF saves the active text file and runs the project compiler. Open PDF launches the compiled output. Download PDF saves just the compiled PDF. Download project package exports the whole project as zip, tar, tar.gz, or tgz.

Troubleshooting

If a PDF does not render, compile once and check the Log panel. If figures do not appear, verify they are inside the project folder and referenced with the correct relative path. Refresh Project reloads files and the PDF from disk.
